Подробное описание

Класс аутентификации клиентов

Предоставляет статические методы для работы с аутентификацией:

  • генерация 64-битной соли в виде 16 hex символов
  • вычисление MD5 хэша по схеме salt+password
  • верификация клиентских учетных данных

Вычисляет MD5 хэш от конкатенации соли и пароля

Аргументы

passwordПароль в открытом виде
Возвращает

Заметки
Порядок: salt + password (соль добавляется перед паролем)
Предупреждения
Для реальных проектов используйте более стойкие алгоритмы (SHA-256, bcrypt)
Аргументы
saltСоль (должна быть 16 hex символов)
passwordПароль в открытом виде
Возвращает
std::string 32-символьный MD5 хэш в hex формате (uppercase)

Формула: hash = MD5(salt + password) Использует OpenSSL библиотеку для вычисления MD5

Предупреждения
Не используйте в продакшене - MD5 уязвим к коллизиям

◆ generate_salt_16()

std::string Authenticator::generate_salt_16 ( )
static

Генерирует 64-битную соль в виде 16 hex символов

Возвращает
std::string 16-символьная hex строка соли
Заметки
Использует случайные числа через std::random_device
Предупреждения
Соль должна быть уникальной для каждой сессии
Возвращает
std::string Соль в hex формате (16 символов, uppercase)

Алгоритм:

  1. Создает генератор случайных чисел на основе std::random_device
  2. Генерирует 64-битное случайное число
  3. Конвертирует в hex строку с заполнением нулями
Заметки
Использует Mersenne Twister для генерации псевдослучайных чисел

Проверяет аутентификационные данные клиента

Аргументы
loginЛогин клиента
received_hashХэш, полученный от клиента
saltСоль, использованная при аутентификации

Возвращает
bool true если аутентификация успешна, иначе false
Заметки
Сравнивает полученный хэш с вычисленным MD5(salt + stored_password)
Исключения
Ничегоне выбрасывает, возвращает false при ошибках
Аргументы
loginЛогин клиента
received_hashХэш, полученный от клиента (32 hex символа)
saltСоль, использованная при аутентификации (16 hex символов)
clientsБаза данных клиентов в формате логин:пароль
Возвращает
bool true если хэши совпадают, false если нет или ошибка

Шаги проверки:

  1. Поиск логина в базе данных
  2. Вычисление ожидаемого хэша: MD5(salt + stored_password)
  3. Сравнение полученного и вычисленного хэшей
Заметки
Возвращает false если логин не найден или хэши не совпадают
