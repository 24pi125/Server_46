Подробное описание

Основной класс TCP сервера

Реализует функциональность многопользовательского TCP сервера:

  • Загрузка конфигурации из файла
  • Настройка и привязка сокета
  • Прием входящих подключений
  • Создание сессий для обработки клиентских запросов
  • Ведение журнала событий
Заметки
Использует блокирующие системные вызовы
Предупреждения
Обрабатывает только одно подключение одновременно (последовательно)
Примеры
/home/penza365/Server_46/src/main.cpp.

void Server::accept_connections ( )

Принимает входящие подключения

Бесконечный цикл, который принимает новые подключения и создает для каждого экземпляр Session для обработки.

Бесконечный цикл, который:

  1. Ожидает входящее подключение (accept)
  2. Получает IP-адрес клиента для логирования
  3. Создает объект Session для обработки подключения
  4. Запускает обработку сессии
Заметки
Обработка блокирующая - следующее подключение ждет завершения текущего
При ошибке accept логирует ошибку и продолжает работу

◆ setup_socket()

void Server::setup_socket ( )
private

Настраивает серверный сокет

Создает сокет, устанавливает опции, привязывает к адресу и порту, переводит в режим прослушивания.

Исключения
std::runtime_errorпри ошибках системных вызовов

Выполняет следующие действия:

  1. Создает TCP сокет (AF_INET, SOCK_STREAM)
  2. Устанавливает опцию SO_REUSEADDR для быстрого переиспользования порта
  3. Привязывает сокет к адресу INADDR_ANY и указанному порту
  4. Переводит сокет в режим прослушивания с очередью на 10 подключений
Исключения
std::runtime_errorпри ошибках системных вызовов (socket, bind, listen)
